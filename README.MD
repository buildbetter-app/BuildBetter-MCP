## Features

### Resources

- **Schema List**: View all available types in your BuildBetter schema
- **Type Details**: Explore fields and structure of specific types

### Tools

- **run-query** â€“ Execute read-only GraphQL (mutations are blocked)
- **list-types** â€“ List all OBJECT types (mutation types are hidden)
- **build-query** â€“ Generate a skeleton query for a type + fields
- **find-fields** â€“ Introspect fields for a given type
- **search-extractions** â€“ Keyword search with optional `type` and `personaIds` filters
- **open-resource / read-resource** â€“ Fetch static docs exposed as resources
- **schema-overview** â€“ Mermaid diagram of core relationships

### Prompts

- **recent-calls** â€“ latest interviews
- **call-details / call-transcript**
- **search-transcript** â€“ phrase search inside a call

## Installation

1. Clone this repository
2. Install dependencies:
   ```
   npm install
   ```
3. Create an `.env` file (or set env vars in your shell) with:
   ```env
   # BuildBetter staging / prod GraphQL endpoint
   BUILDBETTER_ENDPOINT="https://api.buildbetter.app/v1/graphql"
   # Personal API key â€“ generate from BuildBetter dashboard
   BUILDBETTER_API_KEY="YOUR_API_KEY_HERE"
   ```
4. Build the server (compiles TypeScript âžœ ./build):
   ```bash
   npm run build
   ```

## Usage

### Starting the Server

Run the server:

```
npm start
```

### Connecting with Claude for Desktop

Add the server to your Claude for Desktop configuration by editing your `claude_desktop_config.json` file:

```json
{
  "mcpServers": {
     "BuildBetter-MCP": {
      "command": "node",
      "args": ["../BuildBetter-MCP/build/index.js"],
      "cwd": "../BuildBetter-MCP",
      "env": {
        "BUILDBETTER_ENDPOINT": "https://api.buildbetter.app/v1/graphql",
        "BUILDBETTER_API_KEY": "ENTER_KEY_HERE"
      }
    }
}
```
